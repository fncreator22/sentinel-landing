Project 01 — Agent Safety

Every action, reviewed before it runs.

Sentinel sits between an AI coding agent and your machine — checking every command, file write, and git push before it executes. Built fully self-hosted: nothing leaves your computer.

The problem

Agents don't just write code anymore. They run it.

A coding agent that can execute commands can also run the wrong one — a hallucinated rm -rf, a force-push it wasn't asked for, a config change outside the task's scope. Most teams build agents that do things. Sentinel is built to watch them.

How it works

Three checks, cheapest and most reliable first.

01

Rule Engine

Instant, deterministic pattern matching. No model, no ambiguity — known-dangerous commands are blocked immediately.

02

Trained Classifier

A small model trained from scratch on our own labeled data — fast, explainable, and self-hosted.

03

Local LLM Review

Only for genuinely ambiguous cases — runs on a model on your own machine via Ollama. Nothing sent externally.

rm -rf ./build
BLOCK

Matched a hard-blocked pattern. Stopped instantly.

REVIEW

Ambiguous — escalated to the local model for a closer look.

ALLOW

Clearly safe. Executes immediately.

The control panel

Built for people who don't want to touch config files.

Every rule is editable live — including regex patterns — with changes taking effect immediately, no restart. Test any action against the real pipeline before you trust it, register multiple MCP servers, and export your whole setup as a shareable config.

Rule Editor — block/allow patterns, regex support, live saves.

Try an Action — test any command against the real pipeline before you trust it.

Model Settings — local by default; an API fallback is available if you choose it.

MCP Servers — register and monitor multiple guardrail instances.

Export & Share — one click copies a ready-to-paste config for Claude Desktop, Cursor, or CodeX.

Built with

Python FastAPI scikit-learn Ollama SQLite MCP Vanilla JS

No agent framework dependency. Local by default for the model stage — an optional hosted API fallback exists for those who want it, but nothing is required to leave your machine.
